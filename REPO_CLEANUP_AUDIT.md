# MindCraft Repo Cleanup Audit

Last updated: 2026-06-28

Purpose: shared cleanup handoff for Codex, Cursor, and Claude. This repo has months of app, webhook, ML, marketing, and 3D world work. Treat cleanup as engineering hygiene, not a cosmetic sweep.

## Non-Negotiable Rules

- Do not delete tracked source, assets, configs, or generated world files unless the file is explicitly classified in this document or the user approves.
- Before editing or deploying `worlds/world2/`, run `git pull origin main`. Firebase deploys what is on disk.
- Only one agent should push `main` at a time. If another agent pushed first, pull/rebase before pushing.
- Do not commit real `.env`, service account JSON, API keys, local deploy tokens, cache folders, or `node_modules`.
- Prefer path-scoped Git commands for world work, for example `git status --short worlds/world2`.
- If a file is unclear, add it to "Needs Owner Confirmation" instead of removing it.

## Deployment Notes

Firebase hosting targets:

- `app`: public `app/dist`
- `world1`: public `worlds/world2`
- `marketing`: public repo root with many excludes

Important:

- `worlds/world2` is a built static bundle. Editing source-like behavior there often means editing bundled JS or replacing texture assets.
- Cache headers exist for some diagnostic assets, but many world assets are long-lived. Cache-busting should be intentional.
- Deploy the Jesse world with `firebase deploy --only hosting:world1` or `npx firebase-tools@13 deploy --only hosting:world1`, not a broad all-target deploy.
- On 2026-06-28, `61` generated texture inspection files were found inside `worlds/world2`; they added about `32M` under the Firebase public folder. They were deleted.

## Recommended Cleanup Phases

1. Stabilize Git hygiene.
   - Keep the new ignore rules.
   - Delete generated root texture artifacts after approval.
   - Clean Git garbage via `git gc --prune=now` after approval.

2. Security pass.
   - Remove `VITE_GEMINI_API_KEY` from frontend examples unless intentionally public.
   - Verify `app/.env.production` contains no private values.
   - Confirm all real webhook secrets are in Vercel env vars only.

3. Asset ownership pass.
   - For each large asset, mark as production, source-only, archive, or delete.
   - Decide whether world source maps and original textures are deployed intentionally.

4. CSS/design-system pass.
   - Add shared tokens.
   - Migrate repeated colors/radii/shadows incrementally.
   - Avoid visual redesign during cleanup.

5. Workflow simplification.
   - Document one blessed deploy path per surface.
   - Add "pull before world deploy" to agent docs.
   - Keep one-agent-at-a-time push policy for `main`.
