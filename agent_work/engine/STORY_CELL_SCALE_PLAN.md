# STORY_CELL_SCALE_PLAN.md

**Lane:** Engine (`ml/**`, `ml/data/story_cells/`)
**Author:** Fable 5 (Build window, 2026-07-08)
**Implementer:** Cursor or Codex — Engine lane. No `app/**` changes (the merge step at the end touches `app/src/data/storyCells.json`, but that is a data file, not application code).
**Status:** Open

This plan covers scaling story cells from 13 (current) to 200+. A "story cell" is a richer question format than the standard bank: one question, one narrative wrapper (storyContext ≤ 120 chars), one distractor taxonomy (3 wrong answers with misconception IDs), one world_feedback string, 3-step hints, and correct_reasoning — all governed by STORY_CELL_NARRATIVE_RULES.md and the gating system in STORY_INTELLIGENCE_SPEC_V2.md.

**Read before working:**
- `ml/data/story_cells/STORY_CELL_NARRATIVE_RULES.md` — field budgets, cover test, dual-presentation rules
- `PIPELINE_MCQ_SPEC.md` — triple-verify architecture (prevents bad answer keys)
- `CLAUDE.md §Question bank` and `§Active workstream` — bank context
- `ml/scripts/promote_questions.py` — the world_feedback_generator pipeline (shared module target)

### Full cell schema

A full story cell has these fields (from `batch_ingredient_fable5.json`):
```
id, generator_version, title, world, primitive, conceptId, ingredient_id,
misconception_id, misconception_label, level, format, examTag,
storyContext, narrative_need, student_action,
question, choices[], correctIndex, correct_reasoning,

queue_id
```

The `presentation.minimal` field (STORY_CELL_NARRATIVE_RULES.md §3) is **not yet on existing cells but is required on all new cells** from this plan forward.

### The DNA → full cell gap

`batch_llm_002.json` cells were generated by `story_cell_studio.py`. They have all math fields (`question`, `choices`, `correctIndex`, `correct_reasoning`, `distractor_taxonomy`, `hints`, `explanation`) but were missing `ingredient_id` and `gate_status` at generation time (the meta note says to backfill them). The batch_ingredient_fable5.json cells were authored/completed manually in Fable 5 review.

The key generation gap is: the generator produces the math spine correctly, but:
1. `world_feedback` on the top-level cell (the "celebrate the right move" string) needs Fable 5 narrative voice — it's a Katha-voice deliverable, not a math description
2. `distractor_taxonomy[*].world_feedback` (one per wrong choice, ≤ 200 chars each) also needs Fable 5 review
3. `pedagogy_score.math_integrity` must be verified by re-solving (Gate A)

The bulk generator below handles (1) and (2) via LLM with explicit Katha-voice prompting and (3) via auto-verify re-solve.

## B. Bulk Generation Pipeline

### Script

**File:** `ml/scripts/generate_story_cells.py`

This script takes the DNA cells from `batch_ingredient_fable5.json` as the template and generates full story cells for new concepts in bulk. It does NOT modify existing approved cells.

**Usage:**
```
python3 ml/scripts/generate_story_cells.py \
    --concepts number_properties,area_volume,sequences_series \
    [--cells-per-concept 3] \
    [--out ml/data/story_cells/batch_generated_{date}.json] \
    [--no-llm]    # generate math spine only, skip narrative/world_feedback
    [--dry-run]   # validate prompt templates without calling LLM
```

**Dependencies:** Groq LLM (via `LLM_PROVIDER=groq` in `ml/.env.local`). Uses `base.LLMClient` from `ml/scripts/pipeline/base.py` — already provider-agnostic.

### DNA cell format (input)

For each new concept, the script needs a "DNA cell" — a lightweight spec that seeds the LLM with:
- `conceptId`, `ingredient_id`, `misconception_id`, `misconception_label`
- `world` (which story world to use; default to `market_world` if unspecified)
- `primitive` (what math situation anchors the story; e.g. `trade_price_bargain`, `building_plan`, `timing_schedule`)
- `level` (1, 2, or 3)
- `examTag`

DNA cells live in `ml/data/story_cells/batch_ingredient_fable5.json` for the existing 13. **New DNA cells for the priority concepts must be authored first** (see Section C for the priority list). The script should accept an `--dna-file` argument defaulting to `batch_ingredient_fable5.json`; to generate new concepts, pass a separate DNA file.

### Generation pipeline (per cell)

The pipeline is a four-step chain. Each step can fail independently; the script retries up to 2 times per step before skipping the cell.

**Step 1 — Math spine generation**

One LLM call: generate `question`, `choices[4]`, `correctIndex`, `correct_reasoning`, `distractor_taxonomy[3]`, `hints[3]`, `explanation`, `transfer_question`.

Prompt template:
```
You are authoring a math question for a 15-16 year old student (ACT/GCSE level).

Concept: {concept_name}
Ingredient: {ingredient_label} — {ingredient_description}
Target misconception: {misconception_label}

World context: {world} — {world_setting_line}
Primitive: {primitive}

Requirements:
1. Write a 4-choice MCQ where:
   - correctIndex is the index of the ONLY correct answer (0-3)
   - choices[0..3] are all plausible; exactly ONE is correct
   - The question tests the specific ingredient listed above
   - The wrong choices each reflect a specific, named error type (not "random wrong number")
2. math_integrity = 10 is MANDATORY. Verify your answer step-by-step before outputting.
3. The question must be FULLY self-contained. Do not require the story context to solve it.
4. Distractor taxonomy: for each wrong choice, name the exact misconception (student_thinking ≤ 60 chars).

Output JSON only (no markdown wrapper):
{
  "question": "...",
  "choices": ["...", "...", "...", "..."],
  "correctIndex": 0,
  "correct_reasoning": "Step-by-step...",
  "distractor_taxonomy": [
    {"choice_index": 1, "error_type": "...", "student_thinking": "..."},
    {"choice_index": 2, "error_type": "...", "student_thinking": "..."},
    {"choice_index": 3, "error_type": "...", "student_thinking": "..."}
  ],
  "hints": ["Hint 1 (step towards method)...", "Hint 2 (partial compute)...", "Hint 3 (almost there)..."],
  "explanation": "Full worked solution...",
  "transfer_question": "A related problem in a different context..."
}
```

**Step 2 — Math integrity auto-verify (Gate A)**

Re-solve the question independently with a second LLM call that does NOT see the `correctIndex` from Step 1:

```
Solve this multiple choice problem. Show step-by-step work.
End with: "FINAL ANSWER: [letter A/B/C/D]"

{question}
A: {choices[0]}
B: {choices[1]}
C: {choices[2]}
D: {choices[3]}
```

Parse the `FINAL ANSWER:` line. Convert letter to index (A→0, B→1, C→2, D→3). Compare to `correctIndex` from Step 1.

- If they **agree**: `math_integrity = 10`, proceed.
- If they **disagree**: log the cell ID and both solutions, **discard the cell** (do not retry). The discarded cell is written to `ml/data/story_cells/discarded_{date}.json` for human review.
- Do NOT ask the LLM to resolve disagreements — the triple-verify principle from PIPELINE_MCQ_SPEC.md applies here: a single re-solve that disagrees = reject.

This step is the firewall. A wrong key in a story cell corrupts the mastery graph just as much as a wrong key in the standard bank.

**Step 3 — Narrative generation**

One LLM call: generate `storyContext`, `narrative_need`, `student_action`, `world_feedback`, `title`, `distractor_taxonomy[*].world_feedback`, `presentation.minimal.*`, `tone_flags`.

Prompt template (Katha voice):
```
You are writing in Katha voice (the story layer of MindCraft — see brand rules below).

World: {world_name} — {world_setting_line}
Protagonist: {protagonist_name} — {protagonist_bio}
Math move in this scene: {ingredient_label} ({correct_reasoning first sentence})
Student's error this question catches: {misconception_label}

Katha voice rules:
- Present tense, sensory, specific
- One person, one crisis, one mathematical stake (implicit — never name the math)
- No interface words: "concept", "practice", "level", "question", "solve", "calculate"
- storyContext: ≤ 120 chars. Protagonist + stake only. No backstory.
- world_feedback: ≤ 200 chars. One string. Celebrate the move, land ONE math insight.
  - Never: "wrong", "incorrect", "mistake", "try again"
  - Do: name what the student was trying to do + the one corrective move
- presentation.minimal.storyContext: ≤ 60 chars, no character names, plain language

Output JSON only:
{
  "title": "≤8 words",
  "storyContext": "...",
  "narrative_need": "...(≤100 chars, verb-first, story stake)...",
  "student_action": "...(≤80 chars, math verbs allowed)...",
  "world_feedback": "...",
  "distractor_world_feedback": {
    "1": "...(≤200 chars for choice_index 1)...",
    "2": "...",
    "3": "..."
  },
  "presentation": {
    "minimal": {
      "storyContext": "...(≤60 chars)...",
      "world_feedback": "...(≤200 chars, no character names)..."
    }
  },
  "tone_flags": []
}
```

Validation after this step (in Python, before Step 4):
- `len(storyContext) > 120` → revise-loop (re-prompt with "storyContext was {n} chars — compress to ≤120 chars while keeping protagonist + stake")
- `world_feedback` is a dict (not string) → hard reject
- Any tone_flag keyword in NARRATIVE_RULES.md §1.2.3 detected → log warning, continue

**Step 4 — Pedagogy scoring + gate**

One LLM call: assign `pedagogy_score` across 5 dimensions. Use the same scoring prompt template as `story_cell_studio.py` (if it exists) or define:

```
Score this story cell on 5 dimensions (each 1-10):
- math_integrity: Is the answer key definitely correct? (Must be 10 given your verify step)
- diagnostic_power: Will a student who holds the target misconception predictably miss this?
- cognitive_load: Is the narrative load minimal? (≥8 = one-line stake + data in question)
- agency: Does the student feel the stakes, or are they just computing?
- narrative_integration: Is the math necessary in the story, or decorative?

Cell:
{full_cell_json}

Output JSON: {"math_integrity": N, "diagnostic_power": N, "cognitive_load": N, "agency": N, "narrative_integration": N}
```

Gate thresholds (from STORY_INTELLIGENCE_SPEC_V2.md if it exists, otherwise use these defaults):
- **Gate A (auto):** `math_integrity >= 10` AND `diagnostic_power >= 7`
- **Gate B (review):** all of `cognitive_load >= 8`, `agency >= 7`, `narrative_integration >= 7`

Cells that pass Gate A but fail Gate B are written to `batch_generated_{date}.json` with `gate_status: "needs_narrative_review"` — Fable 5 reviews and either approves or requests a reskin. Cells that fail Gate A are discarded (same discard file as Step 2).

### Output file format

`ml/data/story_cells/batch_generated_{date}.json`:
```json
{
  "_meta": {
    "pipeline": "generate_story_cells",
    "generator_version": "scs-v2.0",
    "generated_at": "...",
    "total": 45,
    "passed_gate_a": 40,
    "passed_gate_b": 32,
    "discarded": 5,
    "needs_narrative_review": 8
  },
  "cells": [ ... ]
}
```

Cells inherit `gate_passed: "A"` or `gate_passed: "B"` per STORY_INTELLIGENCE_SPEC_V2.md conventions.

## D. world_feedback Generator Integration

### The shared module

Both the story cell generator (this plan) and the Eedi question enrichment pipeline (`promote_questions.py`) generate `world_feedback` strings in Katha voice. Today these are generated ad-hoc. To ensure voice consistency, create a shared module:

**File:** `ml/mindcraft_graph/world_feedback.py`

This module exposes one function:

```python
def generate_world_feedback(
    concept_id: str,
    ingredient_label: str,
    misconception_label: str,
    student_thinking: str,
    error_type: str,
    story_context: str | None = None,   # for story cells; None = generic
    world: str | None = None,            # story world id; None = generic
    protagonist: str | None = None,
    llm_client: LLMClient | None = None  # from base.py
) -> str:
    """
    Returns a world_feedback string (≤200 chars, Katha voice) or raises ValueError.

    Called by:
    - generate_story_cells.py (Step 3, narrative generation)
    - promote_questions.py (world_feedback fill for Eedi questions)
    
    Cached: ml/data/.world_feedback_cache.json keyed by
    sha1(concept_id + ingredient_label + misconception_label + (story_context or ""))
    """
```

### Shared prompt template

The function uses a single prompt template. Both callers pass different arguments but get the same Katha-voice guarantees.

```
Generate a world_feedback string for a student who chose wrong.

Concept: {concept_id} — {ingredient_label}
Their error: {misconception_label}
What they were thinking: {student_thinking}
{story_line}

Rules (MANDATORY):
- ≤ 200 characters
- Single string (not JSON object)
- Do NOT say: wrong, incorrect, mistake, try again
- DO: name what they were trying to do + the one corrective move
- Katha voice: present tense, one sentence, the move lands
{world_line}

Output the string only, no JSON wrapper.
```

Where `story_line` = `f"Story context: {story_context}"` if provided, else `""`.
Where `world_line` = `f"World: {world} — protagonist: {protagonist}"` if provided, else `"Keep generic (not tied to a story world)."`.

### Cache

`ml/data/.world_feedback_cache.json` (gitignored — add to `.gitignore`). Same pattern as `.explain_cache.json` and `.story_context_cache.json` in the pipeline: dict keyed by SHA1, value is the generated string.

### Updating promote_questions.py

After creating `world_feedback.py`, update `promote_questions.py` to import and call `generate_world_feedback()` instead of leaving `world_feedback: null` in the promotion queue. This closes the main outstanding item in the promote pipeline — after this change, running `--dry-run` will populate world_feedback strings automatically, and Fable 5 reviews the output rather than authoring from scratch.

## E. Merge to App

After story cells are generated, gate-approved, and narrative-reviewed:

```
python3 ml/scripts/merge_story_cells_for_app.py \
    --in ml/data/story_cells/batch_generated_{date}.json \
    --existing-in ml/data/story_cells/batch_ingredient_fable5.json \
    --out app/src/data/storyCells.json
```

`merge_story_cells_for_app.py` already exists. It:
1. Loads all input files
2. Filters to `gate_status: "approved"` cells only
3. Deduplicates by `id`
4. Validates `world_feedback` is a string (not dict) — hard reject if not
5. Validates `presentation.minimal` exists on all new cells
6. Writes the merged output

The output `app/src/data/storyCells.json` is the live file that Practice.tsx and ConceptChapterPage read. Do NOT commit cells with `gate_status != "approved"` to this file.
